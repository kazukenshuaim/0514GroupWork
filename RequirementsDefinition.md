## 1. システム概要

### 1.1. システム構成

```
[ユーザー（ブラウザ）]
        ↓ HTTP (localhost:8501)
[Streamlit フロントエンド]
        ↓ HTTP REST API (localhost:8000)
[FastAPI バックエンド]───────────┐
        ↓                       ↓
[JSONファイル]         [Gemini API]
(ローカルPC)
```

### 1.2. 技術スタック

| レイヤー | 技術 | 用途 |
|---|---|---|
| フロントエンド | Streamlit | 画面表示・ユーザー操作 |
| バックエンド | FastAPI | APIエンドポイント・業務ロジック |
| データ保存 | JSONファイル | 問い合わせデータの永続化 |
| AI推論 | Gemini API | 問い合わせの分類、回答生成 |
| 仮想環境 | uv | Pythonパッケージ管理 |
| バージョン管理 | Git / Sourcetree | ソースコード管理 |
| 開発環境 | Visual Studio Code | ソースコード作成、バックエンドフロントエンドの起動 |

## 2. 機能要件

### 2.1. 問い合わせ入力、送信機能
- 社員が問い合わせ文をテキストで入力する
- 送信ボタン押下でバックエンドにPOSTリクエストを送信する
- 問い合わせ文が空の場合は送信できない（バリデーション）
- 送信成功時、送信成功メッセージを表示
- 4.1. 問い合わせ入力画面にて、この機能を実行

### 2.2. AIによる返事機能
- GeminiAPIに問い合わせを送信する
- 問い合わせ内容をもとに、AIが問い合わせカテゴリ「勤怠・休暇・給与・経費精算・社員情報変更・その他」を自動で分類する
- AIが緊急度を「高・中・低」の3段階で判定する
- AIが問い合わせ内容に対する一次回答を自動生成する
- API接続エラー時、メッセージを表示
- 4.1. 問い合わせ入力画面にて、この機能を実行

### 2.3. 履歴保存機能
- 問い合わせID、問い合わせ内容、問い合わせ日時、カテゴリ、緊急度、回答案をシステムに保存する。IDは問い合わせ順に割り当てられる。
- 4.1. 問い合わせ入力画面にて、この機能を実行

### 2.4. 一覧表示機能
- 2.3. 履歴保存機能にて保存された問い合わせ履歴を一覧表示する
- 表示順は問い合わせ日時の降順（新しい順）
- 表示項目：問い合わせ内容、問い合わせ日時
- 4.2. 一覧画面にて、この機能を実行

### 2.5. 詳細表示機能
- ユーザーが一覧表示から任意の問い合わせを選択することで、システムが詳細を表示する
- 問い合わせIDを指定して表示する
- 表示項目：問い合わせ内容、問い合わせ日時、カテゴリ、緊急度、回答案
- 4.3. 詳細画面にて、この機能を実行

## 3. 非機能要件

### 3.1. 実行環境
- ローカルPC上で動作すること

### 3.2. 操作性
- 機能要件の各機能を、画面遷移３回以内に実現すること

### 3.1. 性能
- JSONファイルの読み書きは問い合わせ件数が1,000件以下の場合に支障なく動作すること

### 3.2. 信頼性
- API接続エラー時、メッセージを表示すること
- JSONファイルが存在しない場合は空のリストとして扱い、アプリが起動できること

### 3.3. セキュリティ
- APIキーは.envで管理し、コードに書かないこと
- .envは、Gitにアップロードしないこと

### 3.4. 保守性
- フロントエンドとバックエンドとJSONファイルは別ファイルで管理すること
- JSONファイルのパスは1か所にまとめる

## 4 画面一覧

### 4.1. 問い合わせ入力画面
- 問い合わせ内容を書くことができる
- 送信ボタンで問い合わせ内容を送信できる
- 送信後、AIからの回答、カテゴリ、緊急度が表示される

### 4.2.  一覧画面
- 問い合わせを一覧で見ることができる
- 問い合わせ内容、問い合わせ日時を、日付の降順（新しい順）に見ることができる

### 4.3. 詳細画面
- 一覧画面に戻ることができる
- 問い合わせ内容、問い合わせ日時、カテゴリ、緊急度、回答案を見ることができる

## 5.API一覧

| メソッド | パス | 機能 |
|---|---|---|
| POST | `/inquiries` |　問い合わせ内容をJSONファイルで保存する　|
| GET | `/inquiries` |　問い合わせ一覧を返す　|
| GET | `/inquiries/{id}` |　指定IDの問い合わせ詳細を返す　|

## 6. データファイル仕様

- ファイル名：`inquiries.json`
- 保存場所：`data/inquiries.json`
- フォーマット：JSON配列（リスト形式）
- 文字コード：UTF-8

```json
[
  {
    "id": 1,
    "created_at": "2026-05-11T10:00:00+09:00",
    "question": "電話番号を変更しました。",
    "answer": "どんまい",
    "category": "社員情報変更",
    "urgency": "中"
  }
]
```
