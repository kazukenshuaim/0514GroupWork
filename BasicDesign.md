## 1. システム構成


## 3. 処理フロー

### 3.1 問い合わせ入力、送信フロー
```
[ユーザー] 問い合わせ内容を入力して送信する
        ↓
[Streamlit] 問い合わせ内容が空でないかチェック（空の場合はエラー表示して停止）
        ↓
[Streamlit] POST /inquiries にリクエスト送信
        |
        | （FastAPI接続エラーの場合、HTTPエラー500を返す）
        ↓
[FastAPI] リクエストを受け取る
        ↓
[FastAPI] IDと問い合わせ日時を生成する
        ↓
[FastAPI] GeminiAPIに問い合わせ内容を送る
        |
        | （GeminiAPI接続エラーの場合、HTTPエラー500を返す）
        ↓
[GeminiAPI] FastAPIから問い合わせ内容を受け取ってカテゴリと緊急度を併せて回答を生成する
        ↓
[FastAPI] GeminiAPIから回答・カテゴリ・緊急度を取得する
        ↓
[FastAPI] JSONファイルが存在しない場合、空のリスト`[]`として続行する
        ↓
[FastAPI] dataフォルダが存在しない場合、新たに作成する
        ↓
[FastAPI] データをinquiries.jsonに追記保存
        ↓
[FastAPI] 保存した1件のデータを Streamlit に返す
        ↓
[Streamlit] 送信成功メッセージを表示する
```
### 3.2 一覧表示フロー

```
[ユーザー] サイドバーで「問い合わせ内容一覧」を選択
        ↓
[Streamlit] GET /inquiries にリクエスト送信
        |
        | （FastAPI接続エラーの場合、HTTPエラー500を返す）
        ↓
[FastAPI] inquiries.json を読み込む
        ↓
[FastAPI] 問い合わせ日時の降順でソートして返す
        ↓
[Streamlit] テーブル形式で一覧表示する
```
### 3.3 詳細表示フロー

```
[ユーザー] 一覧画面で「詳細を見る」ボタンを押す
        ↓
[Streamlit] 選択した問い合わせの ID を保持
        ↓
[Streamlit] GET /inquiries/{id} にリクエスト送信
        |
        | （指定IDが見つからない場合、HTTPエラー404を返す）
        | （FastAPI接続エラーの場合、HTTPエラー500を返す）
        ↓
[FastAPI] inquiries.json から該当IDを検索して返す
        ↓
[Streamlit] 詳細画面に全項目を表示する
```
## 4. データ設計

### 4.1 JSONファイル仕様

| 項目 | 内容 |
|---|---|
| ファイル名 | inquiries.json |
| 保存場所 | data/inquiries.json |
| フォーマット | JSON配列 |
| 文字コード | UTF-8 |
| 初期状態 | ファイルが存在しない場合は空のリスト `[]` として扱う |

### 4.2 データ構造

```json
[
  {
    "id": 1, 
    "created_at": "2026-05-11T10:00:00+09:00",
    "question": "電話番号を変更しました。",
    "answer": "どんまい",
    "category": "社員情報変更",
    "urgency": "中"
  }
]
```
### 4.3 各フィールドの仕様

| フィールド名 | 型 | 生成タイミング | 生成元 |
|---|---|---|---|
| id | 整数 | 保存時にバックエンドで生成 | Python |
| created_at | 文字列（ISO 8601） | 保存時にバックエンドで生成 | Python `datetime.now()` |
| question | 文字列 | ユーザー入力 | Streamlit フォーム |
| answer | 文字列| GeminiAPI作成| バックエンド |
| category | 文字列（勤怠／休暇／給与／経費精算／社員情報変更／その他） | GeminiAPI作成 |バックエンド|
| urgency | 文字列（高／中／低） | GeminiAPI作成 |バックエンド|

## 5. エラー処理方針

| エラーの種類 | 発生箇所 | 対処方法 |
|---|---|---|
| 問い合わせが空 | Streamlit | エラーメッセージを表示し、送信しない |
| dataフォルダが存在しない | FastAPI | dataフォルダを作成する |
| JSONファイルが存在しない | FastAPI | 空のリストとして処理を続行する |
| 指定IDが見つからない（詳細取得） | FastAPI | HTTP 404 を返す |
| サーバー接続エラー| FastAPI | HTTPエラー 500 を返す |
